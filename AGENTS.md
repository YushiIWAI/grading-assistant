# AGENTS.md — Codex 向け指示

# 全プロジェクト共通ルール

## コンテキストの読み込み（最重要）

このプロジェクトには Claude Code 用の `CLAUDE.md` が存在する。
**作業開始時に必ず `CLAUDE.md` を読み込み、そこに書かれたプロジェクト知識・規約・注意事項に従うこと。**

また、以下のファイルが存在する場合はそれも読むこと:
- `.handoff.md` — 前回セッションからの引き継ぎ（あれば最優先で読む）
- `doc-map.md` — プロジェクト内ドキュメントの索引

`CLAUDE.md` に書かれたルール（コーディング規約、命名規則、アーキテクチャ方針など）は
Claude Code 固有の機能に関する記述を除き、すべてこのプロジェクトの正式なルールとして扱え。

## 指示の遵守

- 「全部」「すべて」と言われたら文字通り全部やれ。自分の判断で「これは不要」と間引くな
- 修正を入れる前に「この変更が他のどの操作パスに影響するか」を洗い出せ。影響範囲の分析なしにコードを変えるな

## サジェスト前の安全確認

ツール・サービス・ライブラリ等をサジェストする際は、権利問題・倫理的問題・評判リスクがないか事前に確認すること。特に音声合成・画像生成など権利関係が敏感な領域では、無断学習・無許可利用の懸念がないかを注意する。

## UI変更時のルール（必須）

UI等の画面上の視覚にまつわる部分を変更した場合、以下を必ず実施すること:

### 変更前
- 変更対象の既存デザインのトーン・余白・配色を把握する
- 既存の CSS / デザイントークン / コンポーネントライブラリを確認し、それに合わせる

### 変更時
- 既存のデザインパターン・コンポーネント・変数を優先的に再利用する
- 色・余白・フォントサイズ等はハードコードせず、既存の変数やユーティリティクラスを使う

### 変更後（検証）
1. dev server を立ち上げて動作確認する
2. 重なり・はみ出し・overflow・z-index 異常がないか確認する
3. 画面全体の統一感・バランスが崩れていないか確認する
4. 結果を報告する

## コーディング一般

- 過剰な設計をしない。依頼された変更だけを行い、周辺コードの「改善」を勝手にしない
- コメント・docstring・型注釈は既存コードのスタイルに合わせる。変更していないコードに追加しない
- セキュリティ脆弱性（インジェクション、XSS、OWASP Top 10）を導入しない

## セッション管理

- 作業開始時: `.handoff.md` があれば読み込んで前回の続きから再開
- 作業終了時: 作業状態を `.handoff.md` に記録する（簡潔な再開ガイド、3-5行）
- `.handoff.md` は Claude Code と Codex の両方で共有される引き継ぎファイルである

# プロジェクト固有設定

## プロジェクト概要

採点支援ツール（Python / Streamlit）。Gemini API を使って答案画像を OCR し、ルーブリックに基づいて自動採点する。
詳細は `CLAUDE.md` と `doc-map.md` を参照。

## 技術スタック

- Python（Flask/Streamlit）
- Gemini API（画像解析・OCR・採点）
- 主要ファイル: `app.py`, `scoring_engine.py`, `pdf_processor.py`, `models.py`, `storage.py`

## Codex 固有の注意事項

- このプロジェクトの `CLAUDE.md` にある「セッション管理」のうち、`.claude/skills/` への参照は Claude Code 固有機能なので無視してよい
- Gemini API キーは環境変数 `GEMINI_API_KEY` で管理。コードにハードコードしないこと
