# 法務分析: 匿名化済みデータの個人情報該当性

作成日: 2026-03-21
ステータス: **分析ドラフト（法務確認前）**
目的: 叔母（弁護士）への相談時の論点整理

---

## 結論（先に）

**grading-assistantが送信する匿名化済みデータは、個人情報保護法上の「個人情報」に該当する可能性が高い。** ただし、「個人情報の第三者提供」に該当しても、実際のリスクは極めて低い。

---

## 分析

### 1. 個人情報の定義（法第2条第1項）

個人情報 = 以下のいずれかに該当するもの:
- (1) 特定の個人を識別することができるもの
- (2) **他の情報と容易に照合することができ、それにより特定の個人を識別することができるもの**

### 2. grading-assistantがGemini APIに送信するデータ

| データ | 送信される | 個人を識別できるか |
|--------|-----------|-------------------|
| 生徒の氏名 | **送信しない** | — |
| 学生番号（組・番号） | **送信しない**（仮番号S001等に置換） | — |
| 回答テキスト | 送信する | 通常は識別不可 |
| 採点基準 | 送信する | 個人情報ではない |

### 3. 「容易に照合」の問題

**ここが核心。**

個人情報保護委員会のFAQ（Q1-19）によると:

> 「特定の個人を識別することができる情報に割り当てられている識別子（例：顧客ID等）と**共通のものが割り当てられている**ことにより、**事業者内部において**、特定の個人を識別することができる情報とともに参照することが可能な場合」

grading-assistantの場合:
- サーバー内に「S001 → 1-3（山田太郎）」のマッピングテーブルを保持している
- このマッピングを使えば、Gemini APIに送信した回答テキストを特定の生徒に紐付けることが**容易に**できる
- したがって、**提供元（grading-assistantサーバー）にとっては「個人情報」に該当する**

### 4. 「提供元基準」の適用

個人情報保護委員会は「**提供元基準**」を採用している:
- 提供元にとって個人情報であれば、提供先（Gemini API）にとって個人情報でなくても、「個人情報の第三者提供」に該当する
- つまり、Gemini API側ではS001と回答テキストから個人を特定できなくても、grading-assistantサーバー側でマッピングを持っている以上、法的には第三者提供に該当する

### 5. ではどうすればいいか

#### 選択肢A: 「仮名加工情報」として整理する

仮名加工情報（法第2条第5項）= 他の情報と照合しない限り特定の個人を識別できないように加工した情報。

grading-assistantの匿名化処理は、まさに仮名加工情報の定義に合致する。
仮名加工情報は**第三者提供が原則禁止**（法第41条第6項）。

→ ただし、「委託」に該当する場合は第三者提供に当たらない（法第27条第5項第1号）。

#### 選択肢B: 「委託」として整理する（推奨）

個人情報保護法第27条第5項第1号:
> 「個人情報取扱事業者が利用目的の達成に必要な範囲内において個人データの取扱いの全部又は一部を委託することに伴って当該個人データが提供される場合」は、第三者提供に該当しない。

**つまり、Gemini APIの利用を「採点処理の委託」と整理すれば、第三者提供に該当しない。**

この整理の根拠:
- 利用目的（採点支援）の達成に必要な範囲内でのデータ送信
- Google（Gemini API）はデータを「処理」するだけで、自社の目的で利用しない（DPAで担保）
- Googleとの間にDPA（Data Processing Addendum）が存在し、データ処理者としての義務を負っている

→ **「委託」として整理する場合、本人同意は不要。ただし、委託先の監督義務（法第25条）が発生する。** 具体的には、Gemini APIのDPAの内容が法第25条の監督義務を果たすに足るかの確認が必要。

#### 選択肢C: マッピングテーブルを破棄してから送信する（技術的に不可能）

マッピングがなければ「容易に照合」できないので個人情報に該当しない。しかし、結果を元の生徒に紐付けられなくなるため、サービスとして成立しない。

→ 現実的ではない。

### 6. 法第28条（外国にある第三者への提供）との関係

「委託」として整理した場合でも、委託先が外国にある場合は法第28条の適用が問題になる。

個人情報保護委員会のガイドライン（外国にある第三者への提供編）2-2:
> 外国にある事業者に個人データの取扱いを委託する場合は、法第28条第1項に基づく措置が必要。

具体的には:
- (1) 本人の同意、または
- (2) 基準適合体制の整備（委託先が個人情報保護法と同等の保護水準を確保していること）

**DPA（Data Processing Addendum）が存在し、Googleがデータを学習に使わない・人間レビューしないと約束している場合、(2)の「基準適合体制」として整理できる可能性がある。**

→ ただし、これは法務判断が必要な論点。

## 叔母さんに確認すべきポイント（整理済み）

### Q1: 「委託」整理は成立するか？

「生徒の回答テキストを仮番号に置換してGemini APIに送信し、採点結果を受け取る。Googleとの間にDPA（Data Processing Addendum）があり、Googleはデータを自社目的で利用しない。」

→ これを個人情報保護法第27条第5項第1号の「委託」と整理して、第三者提供に該当しないとすることは可能か？

### Q2: 外国への委託の場合の法第28条対応

→ DPAの存在で法第28条の「基準適合体制」として整理できるか？ それとも保護者の同意（オプトイン）が必要か？

### Q3: そもそも匿名化の程度

→ 氏名もIDも送らず、回答テキストのみ送信。回答テキスト単体で個人が特定される可能性は極めて低い。それでも「容易に照合」に該当するという提供元基準の適用は、このケースで現実的に問題になるか？

### Q4: 保護者同意の要否

→ 上記の整理次第だが、いずれにせよ保護者への説明は行う予定。法的に「同意」が必要なのか、「通知」で足りるのかの判断。
